5.1 Index Lifecycle Management (ILM) Deep


Hot-Warm-Cold-Delete Phases

        Time-based Index Lifecycle

        Hot Phase   -->   Warm Phase   -->   Cold Phase   -->   Delete

        Hot   : Active writes
        Warm : Read-heavy, less writes
        Cold : Rare access
        Delete: Remove old data



ILM automates index movement between data tiers to reduce cost while maintaining performance.


Data Tiers

        Hot Nodes   : Fast SSD, High CPU
        Warm Nodes : Moderate performance
        Cold Nodes : High storage, Low cost
        Frozen Tier : Snapshot-based searchable storage



•   Hot handles ingestion

•   Warm handles search-heavy workloads

•   Cold optimized for cost

•   Delete phase removes expired indices

5.3 Snapshot & Restore


Incremental Snapshot Mechanics

        Snapshot 1 -> Stores Segments A, B, C
        Snapshot 2 -> Stores Only New Segment D

        Snapshots are incremental
        Segments reused across snapshots



Snapshots store immutable segments. Only new segments are copied, reducing storage usage.
