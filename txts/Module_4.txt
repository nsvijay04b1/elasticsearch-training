4.1 Query DSL Structure Deep Dive


Query Context vs Filter Context

        Query Context:
          - Calculates relevance score
          - Used in must / should

        Filter Context:
          - No scoring
          - Faster
          - Used in filter clause



Use query context when relevance ranking matters. Use filter context for exact filtering.


Bool Query Architecture

        Bool Query
        ----------------------------------
        must      -> AND logic
        should    -> OR logic
        filter    -> No scoring
        must_not -> NOT logic
        ----------------------------------


        Example:

        GET /products/_search
        {
          "query": {
            "bool": {
              "must":   { "match": { "name": "running shoe" }},
              "filter": { "range": { "price": { "lt": 100 } }}
            }
          }
        }

Analyzers & Tokenizers

        Tokenizer:
          running
          shoes

          running
          shoes




Stemming

        run, running, runs -> stemmed to "run"




Synonyms

        "sneakers" => "shoes"



Analyzers define how text becomes searchable tokens.


4.3 BM25 Ranking Explained


•   Term Frequency (TF) – How often term appears in doc

•   Inverse Document Frequency (IDF) – How rare term is

•   Field Length Normalization – Penalizes long fields

        Higher score if:
        - Term appears many times
        - Term is rare in corpus
        - Field length is reasonable



You do not need mathematical formulas while teaching. Explain conceptually.


4.4 Aggregations Framework


Bucket vs Metrics

        Bucket Aggregation:
          Group documents (terms, date_histogram)

        Metrics Aggregation:
          Calculate values (avg, sum, max)


        Example:

        GET /sales/_search
        {
          "size": 0,
          "aggs": {
            "by_category": {
              "terms": { "field": "category.keyword" }
            }
          }
        }




Pipeline Aggregations

Operate on results of other aggregations (moving_avg, derivative).


4.5 ES|QL vs EQL vs SQL

        SQL -> Structured relational-style queries
        EQL -> Event sequence queries (security use cases)
        ES|QL -> Pipeline-based analytics



Choose based on use case: analytics, event correlation, or SQL familiarity.


4.6 Search Optimization Techniques


•   Use filter context for non-scoring queries

•   Use keyword fields for aggregations

•   Avoid heavy scripts in queries

•   Monitor slow logs

•   Proper shard sizing



Caching

Filter queries can be cached. Frequently used filters improve performance.


Doc Values

Doc values are columnar storage used for sorting and aggregations.
