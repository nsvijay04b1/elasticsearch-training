6.1 REST API Design Philosophy

        HTTP Methods Used:

        GET      -> Retrieve
        POST     -> Create/Search
        PUT      -> Create/Replace
        DELETE   -> Remove



Elasticsearch is REST-first. Every operation is accessible via HTTP. Cluster state, index
management, search, updates — all exposed via APIs.

        GET /_cluster/health
        GET /index/_search
        PUT /index
        DELETE /index



This makes Elasticsearch language-agnostic and easy to integrate.

6.3 Painless Scripting Internals

Painless is Elasticsearch’s secure scripting language. It runs inside a sandbox.

        Use Cases:
        - Update documents
        - Custom scoring
        - Scripted fields


        POST /products/_update/1
        {
          "script": {
            "source": "ctx._source.price += 10"
          }
        }



•   Compiled to bytecode

•   Sandboxed execution

•   Avoid heavy scripts in queries
