# Lab 10.2: Building Kibana Dashboards

## Goal
Use the Kibana UI to create interactive visualizations and assemble them into a dashboard, transforming raw Elasticsearch data into actionable charts.

## Scenario
Your manager wants a visual summary of the product catalog: a pie chart of products by category, a bar chart of prices, and a data table. Instead of writing JSON queries, you will use Kibana's point-and-click interface.

## Prerequisites
- Completion of Lab 6 (The `products` index must exist with 8 sample documents).
- Kibana must be running and accessible via your web browser.

---

## Part 1: Create a Data View (Index Pattern)

Before visualizing data, Kibana needs to know which index to query.

### 1. Navigate to Data Views
- Open Kibana (`http://localhost:5601`).
- Click the **☰ hamburger menu** → **Stack Management** → **Data Views**.

### 2. Create a New Data View
- Click **Create data view**.
- **Name**: `products`
- **Index pattern**: `products`
- **Timestamp field**: Select "I don't want to use the time filter" (our products don't have timestamps).
- Click **Save data view to Kibana**.

---

## Part 2: Create Visualizations

### Visualization 1: Pie Chart — Products by Category

1. Click **☰** → **Analytics** → **Dashboards** → **Create dashboard** → **Create visualization**.
2. In the visualization editor:
   - **Visualization type**: Select **Pie** from the dropdown (top-left).
   - **Data view**: Select `products`.
   - **Slice by**: Drag the `category.keyword` field to the **Slice by** area, or click **Add** → **Terms** → `category.keyword`.
   - **Size**: Leave as **Count** (counts documents per category).
3. You should see a pie chart with 3 slices: Footwear, Accessories, Apparel.
4. Click **Save and return**.

### Visualization 2: Bar Chart — Price Distribution

1. From the dashboard, click **Create visualization** again.
2. In the visualization editor:
   - **Visualization type**: Select **Bar vertical**.
   - **Horizontal axis**: Click **Add** → **Terms** → `name.keyword` → Set **Number of values** to `10`.
   - **Vertical axis**: Click **Add** → **Average** → `price`.
3. You should see a bar chart showing each product name with its price.
4. Click **Save and return**.

### Visualization 3: Data Table — Full Product List

1. Click **Create visualization** again.
2. In the visualization editor:
   - **Visualization type**: Select **Table**.
   - **Rows**: Click **Add** → **Terms** → `name.keyword`.
   - Add another row: **Terms** → `category.keyword`.
   - **Metrics**: Click **Add** → **Average** → `price`.
3. You should see a table with product names, categories, and prices.
4. Click **Save and return**.

---

## Part 3: Save the Dashboard

1. You should now see all 3 visualizations on your dashboard.
2. Click **Save** in the top-right corner.
3. **Title**: `Product Catalog Dashboard`
4. Click **Save**.

You now have a fully interactive dashboard! Try clicking on a pie slice (e.g., "Footwear") — notice how the other charts automatically filter to show only Footwear products. This is called **cross-filtering**.

---

## Part 4: Discover (Bonus)

Explore the raw documents underlying your visualizations:

1. Click **☰** → **Analytics** → **Discover**.
2. Select the `products` data view.
3. You should see all 8 documents listed with their fields.
4. Try typing a KQL query in the search bar:
   ```
   category : "Footwear" and price > 100
   ```
5. Only "Walking Shoe" ($120) and "Hiking Boots" ($180) should appear.
